OAuth2.me

DOMAIN NAME INDEPENDENCE

WHAT IS IT?

Using a branch-per-feature git flow, you might have a trouble with quality assurance and testing, if your project is closely connected with OAuth 2 providers like Google, Facebook etc). In such a case you need to update the list of callbacks manually to let your QAs to login into the separate branches and test the features.

We created OAuth2.me service to set the callback url only once.

How to use it?

  1. Set your redirect uri to http://oauth2.me/redirect
  2. Your application should use the state parameter to pass your redirect URI encoded with base64
We've created a ruby gem for easy integration with your Ruby/Rails project: https://github.com/jetthoughts/oauth2-me-ruby



How to install your own OAuth2.me?

  1. Make a fork of our repository
  2. Remove the `CNAME` file from the root.
  3. Wait until github will compile the website for you.

Important

Do not use OAuth2.me service on your production server. It is using the state parameter to redirect you to right server.
Normally, you should use it to protect your application from CSRF attacks.
